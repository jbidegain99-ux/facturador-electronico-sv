# Audit Report

## Executive Summary
This monorepo is not audit-ready. The core problems are structural, not cosmetic: the backend concentrates legal/compliance logic, persistence, and side effects in 2 very large services; the frontend bypasses its own shared data/auth abstractions at scale; Docker/local-dev config is internally inconsistent; and sensitive operational data is over-logged and partially committed.

Measured scope:
- 2.6 GB working tree; `apps/web/.next` alone is 787 MB, root `node_modules` 1.1 GB, `apps/api/coverage` 4.0 MB.
- 61 `page.tsx` files, 58 `*.service.ts` files.
- Largest files: [schema.prisma:1](/home/jose/facturador-electronico-sv/apps/api/prisma/schema.prisma), 1,813 LOC; [dte.service.ts:1](/home/jose/facturador-electronico-sv/apps/api/src/modules/dte/dte.service.ts), 2,505 LOC; [hacienda.service.ts:1](/home/jose/facturador-electronico-sv/apps/api/src/modules/hacienda/hacienda.service.ts), 1,167 LOC; [catalogo/page.tsx:1](/home/jose/facturador-electronico-sv/apps/web/src/app/(dashboard)/catalogo/page.tsx), 1,629 LOC; [facturas/nueva/page.tsx:1](/home/jose/facturador-electronico-sv/apps/web/src/app/(dashboard)/facturas/nueva/page.tsx), 1,163 LOC.
- Frontend duplication counts: 180 direct `localStorage.getItem('token')` reads and 241 direct `process.env.NEXT_PUBLIC_API_URL` reads in `apps/web/src`.
- Type escapes: 58 `any` references in `apps/api/src`, 16 in `apps/web/src`.
